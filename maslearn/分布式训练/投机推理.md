投机推理（Speculative Decoding）是一种高效的推理方式，可提升推理性能（如ITL）、降低推理成本，在特定场景下能够提升用户体感。目前投机推理已成为了一种标准的解决方案，常见的有：

草稿模型。给大模型配置一个草稿（draft）模型，草稿模型负责字符生成、大模型验证生成结果。
辅助层。在大模型基础上增加一些头层，使其能生成更多的字符，如Medusa、EAGLE等。
数据匹配。利用历史数据/模板，通过规则匹配的方式生成字符，然后用大模型进行校验，如ngram、suffix decoding。
投机推理也可以叫做推测解码、投机采样（Speculative Sampling），本文中统一用“投机推理“。

1 方案介绍
1.1 基本原理
投机推理是一个猜测、验证、比较的过程，结合概率统计知识先介绍其基本原理。

猜测大模型的输出。目前主流的大语言模型（LLM）采用的是自回归生成的方式，是一种依赖采样（Dependent Sampling），其概率用
表示，特点是历史的采样值影响当前采样输出。

当使用“猜测”的方式去预测大模型输出时，存在如下特征：

n越大预测序列偏移量越大；
采样的确定性越高（随机性低），预测容错空间越小。
第1点可理解为猜测得越多，可能导致输出质量越差；对于第2点的理解，举个例子说明。如一个填词题：

“平\_\_”？

可能的结果有“平凡“、”平方”、“平齐”、“平行”等。假设让LLM模型推测一个结果，用概率最高值作为输出(比如“平行”)，我们随机猜测一个结果与之相等的概率是：1/t，其中t表示可能结果。

为了增加1/t数值，要么扩大其分子，要么缩小其分母。如，让模型输出多个值（topK结果都算正确），提升猜测值与模型输出值的重叠量。

可以用于去猜测大模型输出的方法有多种，而实践中希望找到一个让1/n更大的方案。

模型多输出值的作用。在LLM推理中，当输入一个prompt时可得到同长度的logits输出，单个logits值可用于预测下一个字。自回归中只取logits最后一个值，前L-1个logits丢弃（要么不生成），因为前L-1个字符都是已知的。如果解析前面的logits，对应的预测值能否与输入的prompt字符对上吗？

答案是：不一定。 因为模型的输出是概率分布，只能说预测到原字符的概率更大。比如下图中，我们取第三个字符‘k’的输出logits并采样，结果可能不是‘a’，出现了LLM的预测值与输入值不相等的情况。

因为前L个字符是已知的，所以计算L-1字符的采样输出对于LLM生成L之后的字符来说没有意义。但是却能利用LLM支持多输出的能力校验（评估)另一个的模型的输出质量。

直观一点就是，让模型A生成了一串字符，把该字符输入到能力更强模型B评价一下这个输出是否达标。

评价过程。要对预测的结果好坏进行评估，其一般规则：当猜测结果的概率高于模型输出的概率时，则认为预测是可信的，反之亦然。举个例子，如下图所示，概率分布A是小模型的预测输出，概率分布B是主模型标准输出。若小模型预测出了‘n’值，‘n’在两个模型中的概率分别为0.37、0.67，因为0.37 < 0.67，则‘n’值被接受。

若把大模型比作师父，小模型比作徒弟：

猜测大模型的输出：徒弟模仿师父的动作；
模型多输出值的作用：师父通过自身能力指点徒弟；
评价过程：徒弟的工作需要得到了师父的认可；如果徒弟出错了，师父就纠正。
1.2 草稿模型
带有草稿（draft）模型的投机推理过程一般是先用草稿模型生成数据，然后用参数量较大的模型进行结果验证/生成。结合下图来介绍该方法，假设有两个模型LLM-1和LLM-2，LLM-1负责输出、LLM-2负责校验。基本步骤：

输入提示词prompt到LLM-1，经过n轮递推迭代后生成对应结果。
将输出结果传递给LLM-2计算，得到对应的logits。
用单个logits计算得到下一个字符概率分布，i’->‘y’预测概率，若‘y’被接受，则进行下一步迭代，依次类推。
若步骤3中字符被拒绝，则让采样回退一步，并返回结果(或者重采样)。

阈值比较：设定一个阈值，如果预测概率小于阈值，则接受该字符并继续预测；如果概率大于阈值，则丢弃该字符和后续所有值，并重新预测。输出格式：

小模型生成2个字符+大模型生成1个字符
在LLM-2的校验运算中，更常用的方式是因果掩膜（Causal Mask）。对比一般解码（Decode）过程[2]，投机推理增加了草稿模型的解码输出，查询（Query）长度为草稿模型长度加1，即L=1+n。

过程对比示意
草稿模型方案的优点：

草稿模型一般会选择开销小的模型（比如用0.1倍的参数）[3]，相比大模型直接生成速度更快；

一般推理与投机推理的对比
单次可生成n个字符；最坏的情况是第一次校验就失败，但至少能生成一个字符；
接受阈值可以作为一个调节超参，满足不同场景的需求。
不足：

小模型能力不足，或者参数选取不当可能会导致输出质量下降；
如果首次验证就失败，模型的计算量比不用投机推理的计算量更大。
1.3 辅助层
与草稿模型原理类似的方式还有另一种：在主模型上额外添加一些层来进行多字符预测，这些层和主模型同时训练、共享主模型信息。跟草稿模型方案最大差异点是这种方式的大模型与辅助层之间存在耦合关系。增加辅助层的方案也有多种，比如Medusa[4]、EAGLE[5]、MTP[6]、MLP speculator[7]等，选取两个典型方法介绍：

EAGLE(Extrapolation Algorithm for Greater Language-model Efficiency)的核心思想利用主模型的中间输出（LM Head的输入）信息提供给草稿模型进行自回归生成token。关键点：

辅助模型的Embedding和LM Head共享参数；
递推需要利用主模型（target LLM）的中间信息，这些特征值f传递给辅助模型层计算；
在迭代的过程中f与embedding的输出e进行拼接作为辅助层主体的输入；
这种方式下，辅助层的forward输入宽度可以调节。

EAGLE-1的推理过程
EAGLE目前进行了两个版本的迭代，最新的是EAGLE3（参考：https://arxiv.org/abs/2503.01840）

MTP(Multi-Token Prediction)思想与EAGLE类似，其结构也是在主模型层的后面接几个头层，不过MTP更好的适配了训练场景，且随着DeepSeekV3发布受到了更多关注。

结构如下图所示，主模型（Main Model）后面接的MTP Module1/2可视作多个小模型。在一次推理中可生成多个token，图中t5是主模型生成的数据，而t6、t7...由MTP生成。

在推理中MTP模块本轮生成的token不会直接作为输出返回，而是先保存为草稿。在下一轮推理中，通过校验确定是否被接受。举个例子，若模型本轮生成了：t5、t6、t7...，而推理返回值为t5；下一轮中main model的输入为t5~t7...，采用直接概率比较方式，如果阈值范围内能预测出t6，则接受t6，依次类推。

MTP推理过程
对框架（如vLLM）的影响：

1、投机推理不会改变推理框架调度逻辑，依然保持单次调度完成相同数量的前向计算。

2、推理框架的缓存KV cache逻辑需要适配：

kv cache缓存的tokens数量增加上投机推理的空间，比如MTP中有几个头就要多开启几个tokens地址位置；
当tokens未被采纳时，对应的kv cache得进行回退。常用的方式是直接覆盖。
1.4 数据匹配预测
在处理一些高频重复文本的场景中，如法律、规则等，可以用数据规则匹配预测LLM的输出。常见的方式有Ngram、Suffix decoding。

Ngram（gram指代元组)：是一种基于历史数据进行概率计算并获得预测结果的方式。其核心的思想是：一个词出现的概率依赖于它前面的 N-1 个词。举个例子，假设输入的前缀为“我要吃”，可能的尾缀有“苹果”、“西瓜”，若对应的概率值为：

西
瓜
我
要
吃

苹
果
我
要
吃

则输出为“苹果”，这个Ngram例子中匹配采用了三个字，输出预测了两个字。

在推理中应用Ngram方法，一般步骤：

确定匹配长度、预测长度。
根据已有数据(prompt或者历史数据)构建Ngram词表。
用主模型验证结果，若满足则接受，反之亦然。
Suffix decoding尾缀匹配：根据输出数据来构造尾缀树，匹配到合适的尾缀后选择得分最高的值作为输出，得分的高低与数据出现频率相关。suffix decoding涉及有两个关键参数：窗口大小P、后缀节点数量N。

结合下图示例（P=2, N=3），输入为“85485”，suffix decoding匹配的头有两种“5”和“8，5”。

其基本步骤：

根据保存的历史数据，构造第一个后缀树。
根据当前输出构造的另一个后缀树。
找出两棵后缀树中满足要求的子树。
对子树进行打分排序，挑选出分数最高的子树。
用LLM验证子树的值是否满足要求，若满足则接受。

数据匹配的优点：

计算量小、不需要额外显存空间；
极大的降低了ITL，输出速度更快；
因为无草稿模型，不用同时协调两个模型的部署，框架适配更加方便。
不足：

仅在一些文本重复率比较高的场景使用，场景变化比较大时，接受率低。
2 投机推理的使用示例
目前投机推理基本功能在主流框架（vLLM/SGLang）中均已适配，以vLLM 0.9.x的离线场景来列举几个投机推理的使用方法。

关键参数：

speculative_config：投机推理配置，包括小模型参数。
num_speculative_tokens：草稿模型输出的tokens的数量。
method：计算方式，如ngram、mtp、eagle、eagle3；
prompt_lookup_max：最大的查询长度，用于ngram。
prompt_lookup_min：最小的查询长度，用于ngram。
2.1 草稿模型

# 在vLLM中使用投机推理的示例：

from vllm import LLM, SamplingParams

prompts = [
"InfraTech是什么类型的公众号？",
]
sampling_params = SamplingParams(temperature=0.9)

llm = LLM(
model="Qwen/Qwen3-VL-235B-A22B-Thinking",
tensor_parallel_size=1,
speculative_config={
"model": "Qwen/Qwen2.5-VL-3B-Instruct",
"num_speculative_tokens": 5,
},
)
outputs = llm.generate(prompts, sampling_params)

for output in outputs:
prompt = output.prompt
generated_text = output.outputs[0].text
print(f"Prompt: {prompt!r}, Generated text: {generated_text!r}")
注意：草稿模型的功能需要在0.9.x中使用，v0.10.x中使用会报NotImplementedError错误。
