Dim中没有使用的维度，设置为：1；Idx中没有使用的维度设置为：0。2. 使用统一内存降低编写难度
在code编写的初期，可以使用统一内存来降低编写与阅读难度。避免了GPUToHost、HostToGPU的操作，从而快速验证算法（kernel）的正确性，比如：

float *x, *y;
cudaMallocManaged(&x, N*sizeof(float));
cudaMallocManaged(&y, N*sizeof(float));
for (int i = 0; i < N; i++) {
x[i] = 1.0f;
y[i] = 2.0f;
}
int blockSize = 256;
int numBlocks = (N + blockSize - 1) / blockSize;
add<<<numBlocks, blockSize>>>(N, x, y);
代码中在给x y赋值时可直接在主机上进行操作，然后直接把数据代入add kernel中计算。示例代码：um_demo.cu 编译方式“nvcc -lcuda um_demo.cu -o um_demo”运行“./um_demo”。

3.4 event
在编写kernel函数时，我们一般需要知道kernel在GPU端的运行时间，通常使用event来统计时间，而不是使用cpu的timer（统计时间不准确！）。event使用示例如下，其中func为待统计的运算函数：

#define TIME_ELAPSE(func, elapsedTime, start, stop) \
 cudaEventCreate(&start); \
 cudaEventCreate(&stop); \
 cudaEventRecord(start, 0); \
 (func); \
 cudaEventRecord(stop, 0); \
 cudaEventSynchronize(stop); \
 cudaEventElapsedTime(&elapsedTime, start, stop); \
 cudaEventDestroy(start); \
 cudaEventDestroy(stop);
event的使用具体可以参看：定义：memory_opt 30Line ，使用示例zero_copy.cu

4 减少数据的拷贝/换页
如果运算时间主要消耗在数据传输/拷贝（通过工具能检查出来），可以通过共享内存、零拷贝、页锁内存等降低数据传输成本。

零拷贝：当数据保存在主机上，且GPU只需要使用一次时，我们借助零拷贝来实现数据传输。可以避免数据从全局显存的进出，从而提供效率。

例如向量加法运算，当使用零拷贝时，数据吞吐能够极大提高。

示例代码：zero_copy.cu 编译：“$ nvcc -lcuda -I../memory_opt/ zero_copy.cu -o zero_run”，运行“./zero_run”，结果：

> . Data tranfer via zero copy. VectorAdd throughput: 714.285706 GB/s
> 共享内存：用户可直接使用的片上存储。对于需要反复使用的数据，将数据放到共享内存中，因为共享内存的速度与L1 cache相同，相比全局显存效率更高。

5 提升存算重叠度
提升存算重叠度是指让数据操作与运算操作上尽量并发执行，减少相互的等待消耗。比较有效的方法是采用多stream异步运算。 这里我们分开解释一下：

5.1 多stream运算
多stream（计算流）运算是指在启动多个CUDA stream来共同消耗GPU资源完成既定运算目标。通过GPU硬件手册可以知道在GPU卡中数据的拷贝与运算是由不同的器件去完成的。CUDA中给出了这两个抽象单元：拷贝引擎（Copy Engine）和 计算引擎（Kernel Engine），这两个引擎在支持异步运算， 比如我们需要完成矩阵乘法运算C = A x B，一般流程如下所示（假设拷贝与运算都是消耗一个单位时间）：

该示例中运算由单stream完成，数据拷贝与运算是按照代码给定顺序进行，图中两个矩阵运算的也会依次进行，可以看到Copy Engine 工作时Kernel Engine是限制的；如果启动两个stream，在运算C1的时候，能够同时去运输计算C2所需要数据，这样就缩短了整体运算时间。

当然示例中的计算和数据拷贝时间都用了相等的时间来表示，实际中拷贝与运算的速度是不一样的，取决于服务器的硬件能力。

5.2 异步计算
对于同一个运算，我们可以通过构造异步运算来提升整体的计算效率；如下图所示，展示了两个数据相加的运算，我们可以利用多stream拆分运算；然后先让数据**异步入队**再运算；

矩阵运算也能进行异步运算，C = A x B举例，同步的情况下，操作步骤是先把矩阵A数据运送到GPU，然后把矩阵B运送到GPU，再进行计算，意味着如果数据A 和数据B 没有都传输完成前，kernel engine是不会进行计算的。 我们可以通过异步计算的方式来进行矩阵计算，即把矩阵进行拆分计算， 即用子矩阵的运算来实现。 在矩阵的size比较大时，加速效果明显。
