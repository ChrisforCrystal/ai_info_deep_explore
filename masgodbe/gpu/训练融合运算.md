1、计算内容与公式
scaled-mask-softmax是对标准softmax函数的一种扩展，它整合了比例缩放（scaling）和掩码操作（masking）。比例缩放通过将softmax中的每个元素乘以一个固定的系数来改变输出概率分布的陡峭程度；掩码操作则用于在softmax之前处理特定的输入元素，例如将不相关或无用的数据点设置为一个非常小的值（通常是负无穷），以确保它们在之后的softmax运算中的影响被最小化或完全忽略。这种融合操作可以提高GPU上的运算效率，尤其是在深度学习和神经网络的场景中，通常会看到其显著的性能提升。

1.1 参数
输入参数与输出参数的格式:

输入参数： [batches, attn_heads, query_seq_len, key_seq_len]

输出参数： [batches, attn_heads, query_seq_len, key_seq_len]

2. kernel函数设计
   2.1 threads块的设计与坐标计算
   线程块设计：

为了提速求和运算，比如适配CUDA里面的warp操作，线程块block中按照32的倍数组织线程。

一般而言：每个block里面包含的线程threads_per_block推荐值为128；所以，块的定义设计为：dim3 threads(32, 4） 也可以写成三维形式（32, 4，1）。

计算时32个threads 处理一个/多个softmax计算数据。一个函数的调用，要完成batches _ attn_heads _ query_seq_len次softmax运算。

通过数据构建grid，我们可知需要block的总数为：

cnt = batches _ attn_heads _ query_seq_len / warps_per_block

进一步优化：当key_seq_len比较小时，我们可以让一个warp的threads处理多个softmax计算，一般当key_seq_len小于128时，设置次数为2。这样线程块需要下发的次数变为：

cnt = batches _ attn_heads _ query_seq_len / （warps_per_block \* 2）。

这样，grid的维度设计为：dim blocks(query_seq_len/batches_per_block, attn_heads, batches)。

其中batches_per_block是指一个block能够处理几个softmax计算。

对应的代码如下：

int log2_elements = log2_ceil(key_seq_len);  
const int next_power_of_two = 1 << log2_elements;
int batches_per_warp = (next_power_of_two <= 128) ? 2 : 1; // key_seq_len小于128时，一个warp同时处理两组数据
constexpr int threads_per_block = 128; // 默认的threads 数量
int warps_per_block = (threads_per_block / warp_size); // 一个线程块有几个warp
int batches_per_block = warps_per_block _ batches_per_warp; // 一个warp一次处理的softmax数量 _ warp数量
dim3 blocks(query_seq_len/batches_per_block, attn_heads, batches);  
dim3 threads(warp_size, warps_per_block, 1);
坐标计算：

根据上述线程块格式，我们采用3D2D的线程块（参看【CUDA全局坐标计算&Grid/Block/threadIdx映射处理】数据索引计算方式）。一般的线性数据索引方式是每个block中的thread处理数据按照一对一的关系，而此处，需要用一个warp映射一个softmax计算的关系，每个warp包含32个threads，而每个thread拿的数据量跟key_seq_len相关（一次拿取连续四个，ELEMENTS_PER_LDG_STG=4 ），thread拿取数据的映射关系如下图所示：

对于一个warp中每个thread需要拿去的数据个数：

iterations= sequence_length / warp_size。

如果每次能够拿取“ELEMENTS_PER_LDG_STG”个数据，那么拿取的次数

cout =iterations/ELEMENTS_PER_LDG_STG

进一步，我们计算block在Grid中位置：

blockInGrid = blockIdx.x + blockIdx.y*gridDim.x + blockIdx.z*gridDim.x*gridDim.y;
每个block需要处理的数据量：dataPerBlock = key_seq_len * batches_per_warp \* warp_per_block

对于在block中，每个warp之间的数据间距：key_seq_len* batches_per_warp * threadIdx.y

当ELEMENTS_PER_LDG_STG =1 时，对应的数据索引计算公式：

idx = blockInGrid _ dataPerBlock + key_seq_len_ batches_per_warp \* threadIdx.y + thread.Idx;
offset = idx + warp_size;
为了提升IO操作的质量，可以在拿取次数iterations较大时，让thread每次操作多个（ELEMENTS_PER_LDG_STG =4）半精度数据；这样公式进一步变为：

idx = blockInGrid _ dataPerBlock + key_seq_len_ batches_per_warp _ threadIdx.y + thread.Idx _ ELEMENTS_PER_LDG_STG ;
offset = idx + warp_size \* ELEMENTS_PER_LDG_STG ;
通过前面block定义中可知blockDim.y = batches_per_warp ；我们可以定义一个batch单位，每个batch的长度为key_seq_len，这样就能够获得数据的起始位置：first_batch, （注意这个batch的含义与输入数据的batch是两个概念）。softmax的批次总数等于：[batches, attn_heads, query_seq_len] 相乘。

采用batch的计算方式，数据索引代码实现：

// element_count =key_seq_len；ELEMENTS_PER_LDG_STG = elements_per_once
int first_batch = (blockDim.y _ (blockIdx.x + gridDim.x _ (blockIdx.y + gridDim.y _ blockIdx.z))+ threadIdx.y) _ WARP_BATCH;
int local_idx = threadIdx.x;
src += first_batch _ element_count + ELEMENTS_PER_LDG_STG _ local_idx;
边界情况处理：由于warp处理的batch数量可能超过最大batch数量，所以需要在代码中增加防止越界的处理操作。

2.2 数据到本地寄存器&预处理
本地空间：单个线程的本地寄存器需要拿取的数据数量计算：

data_number = WARP_BATCH \* WARP_ITERATIONS

其中：WARP_BATCH：一个warp需要处理多少个softmax；WARP_ITERATIONS：每个线程需要循环的次数；

数据copy：由于数据处于globalmemory，计算时需要将数据复制到寄存器中。计算数据格式是FP16/BF16，可以采用模版函数进行数据copy。当单次copy的数据大小为1时，直接指针赋值；当每次copy的数量为4时，采用float2进行数据转换后copy。 编译器会优化数据的copy效率。

template <>
**device** **inline** void copy_vector<c10::BFloat16, 4>(c10::BFloat16 *dst, const c10::BFloat16 *src) { _((float2_) dst) = _((float2_) src); }

2.3 Shuffle计算
softmax计算中涉及两个操作需要全量数据：一个是sum()，另一个是max()。单个线程只拿了softmax计算部分数据，所以需要有操作完成线程之间的数据共享。CUDA的数据在线程之间共享，可以通过shared memory或者用特殊的指令来完成。当数据不需要频繁读写时，且操作较为简单时，考虑采用warp_shuffle操作 Warp Shuffle Functions。

其中\_\_shfl_xor_sync() 可以进行warp的线程之间的sum和max 计算，实现代码如下：

template <typename T>
**device** **forceinline** T WARP_SHFL_XOR_NATIVE(T value, int laneMask, int width = warpSize, unsigned int mask = 0xffffffff)
{
#if CUDA_VERSION >= 9000
return **shfl_xor_sync(mask, value, laneMask, width); // 实际的操作函数
#else
return **shfl_xor(value, laneMask, width);
#endif
}

// 通过模板ReduceOp来选择 max计算或者add计算。
template <typename acc_t, int WARP_BATCH, int WARP_SIZE, template<typename> class ReduceOp>
**device** **forceinline** void warp_reduce(acc_t\* sum) {
ReduceOp<acc_t> r;
#pragma unroll
for (int offset = WARP_SIZE / 2; offset > 0; offset /= 2) {
#pragma unroll
for (int i = 0; i < WARP_BATCH; ++i) {
acc_t b = WARP_SHFL_XOR_NATIVE(sum[i], offset, WARP_SIZE);
sum[i] = r(sum[i], b);
}
}
}
通过warp_reduce操作完成多线程之间的数据求和或者求最大值操作。

3 辅助代码
3.1 kernel启动代码
kernel运算中threads批数量一般采用2的指数倍，这样能够让GPU发挥好的计算效率。但，对于计算数据本身而言，数据的长度不一定能够被2整除，在本例的softmax计算中数据长度key_seq_len 不一定恰好为2的指数倍。在计算处理时，采用两个数值来表示key_seq_len。

一个是key_seq_len对数后取2圆整值: log2_elements = log2_ceil(key_seq_len)；

另一个是实际key_seq_len长度：element_count = key_seq_len

log2_elements 作为threads线程操作的圆整单位值；element_count 传入kernel中作为计算元素时的上界限：

int log2_ceil(int value) {
int log2_value = 0;
while ((1 << log2_value) < value) ++log2_value;
return log2_value;
}
int log2_elements = log2_ceil(key_seq_len);
const int next_power_of_two = 1 << log2_elements;

// example： log2_ceil(1234) = 11 ; next_power_of_two = 2048
kernel启动时，我们根据log2_elements 的长度，来选择不同的下发模版函数：

// 不将log2_elements直接带入模版中，通过foreinline 编译器可以提升运算速度。
switch (log2_elements) {
case 0: // 1

3.2 计算每个block处理的softmax数量
当给定输入参数时，按照kernel的构建方法，我们能够确定一个block需要算多少个batch的数据：

int get_batch_per_block(int query_seq_len, int key_seq_len, int batches, int attn_heads){
int log2_elements = log2_ceil(key_seq_len);
const int next_power_of_two = 1 << log2_elements;

    int warp_size = (next_power_of_two < C10_WARP_SIZE) ? next_power_of_two : C10_WARP_SIZE;
    int batches_per_warp = (next_power_of_two <= 128) ? 2 : 1;

    constexpr int threads_per_block = 128;
    int warps_per_block = (threads_per_block / warp_size);
    int batches_per_block = warps_per_block * batches_per_warp;

    return batches_per_block;

}
返回的batches_per_block值中的“batches”实际上指代是需要处理的softmax的数量，非输入参数中数据的batches，输入参数中batches实际上未使用。
