# 附录A：AI Infra 大模型 Token 使用量优化 PRD（可落地版）

## 0. 文档信息

- 文档名称：Token 控制平面（Token Control Plane）PRD
- 版本：v1.0
- 状态：评审稿
- Owner：AI Infra
- 协作方：产品、客户端、服务端、Agent 研发、数据分析、SRE
- 日期：2026-03-04

## 1. 背景与问题定义

当前业务形态包含三类入口：

- 类 Cursor 的智能研发产品（IDE 内对话、代码生成/改写）
- ADK 类 Agent 产品（多步骤推理+工具调用）
- 外部模型统一接入（Higress 网关插件承接）

现状痛点：

- Token 成本不可控：长会话、多工具回填、检索片段冗余导致输入 token 快速膨胀
- 质量与成本难平衡：团队常在“强模型高成本”与“便宜模型质量回退”之间反复
- 策略分散：产品、SDK、应用各自做裁剪逻辑，无法统一治理和观测
- 用户无感知：终端用户不知道为何消耗高，也无法直接启用节省能力

目标是将“省 token”从零散技巧升级为平台化能力：用户可一键使用，研发可复用，Infra 可治理。

## 2. 目标与非目标

### 2.1 目标

- 成本目标：在质量可控前提下，整体 token 消耗降低 30%~50%
- 体验目标：P95 首 token 时延不劣化，关键场景质量回退率 < 2%
- 能力目标：形成产品层、SDK层、网关层职责清晰的统一方案
- 治理目标：建立统一埋点、统一评测、统一灰度、统一回滚

### 2.2 非目标

- 不在 v1 解决跨模型全部 Prompt 兼容性问题
- 不在 v1 引入复杂长期记忆图谱，仅做可解释摘要记忆
- 不在 v1 改造全部历史应用，采用“新流量优先+老流量迁移”策略

## 3. 北极星指标与验收指标

### 3.1 北极星指标

- 每个成功任务 token：`TotalTokens / SuccessfulTasks`
- 成本效率：`Cost / SuccessfulTasks`
- 质量保护：任务成功率、用户满意度、人工抽检通过率

### 3.2 关键指标

- 输入 token 降幅
- 输出 token 降幅
- 缓存命中率（前缀缓存、网关缓存、工具缓存）
- 压缩触发率与压缩后保真度
- 上下文裁剪命中率与误裁率
- P50/P95 延迟、超时率、重试率

### 3.3 v1 验收阈值

- 整体 token 降幅 >= 30%
- 关键业务流质量回退率 <= 2%
- 缓存命中率 >= 20%（可按场景分层验收）
- 观测完整率 >= 99%（请求级 token 字段齐全）

## 4. 用户与场景

### 4.1 用户角色

- 终端用户：希望“开关式省钱”，不关心底层细节
- 应用研发：希望复用标准化中间件，不重复造轮子
- AI Infra：希望统一治理和优化策略，不被业务碎片化绑架

### 4.2 高价值场景

- IDE 连续问答与代码改写
- Agent 多步工具调用（检索、执行、汇总）
- 长上下文问答（文档/代码库分析）
- 重复性任务（模板化请求、固定系统提示）

## 5. 总体方案

采用三层承接模型：

1. 产品层：把节省能力做成可配置产品功能，用户可直接用
2. SDK层：预算、裁剪、压缩、缓存 Hint、路由等标准中间件
3. 网关层（Higress）：跨模型统一接入、缓存、限流、观测、历史治理

### 5.1 职责边界

- 产品层承接：
- 省 token 模式（极速/均衡/深度）
- 上下文来源开关
- 自动摘要开关
- 回答长度档位
- 预算提示与可视化

- SDK层承接：
- Token Budgeter（输入/输出/步骤预算）
- Context Packer（评分、裁剪、重排）
- Compressor（历史摘要、工具输出压缩）
- Cache Client（前缀缓存 key 与语义缓存接口）
- Model Router（分类任务走轻量模型）

- 网关层承接（Higress）：
- `ai-proxy`：模型统一代理
- `ai-cache`：精确缓存与语义缓存
- `ai-history`：会话历史托管与 TTL
- `ai-token-ratelimit`：按租户/应用配额
- `ai-statistics`：token/时延等观测
- `ai-intent`：意图识别与路由增强

- 应用研发最佳实践承接：
- Prompt 结构化模板
- 工具输出字段白名单
- 场景化预算参数
- A/B 验证与回滚脚本

## 6. 产品需求（用户可直接使用）

### 6.1 功能一：省 Token 模式

- 模式定义：
- 极速：优先成本，短输出，强裁剪，轻量模型优先
- 均衡：默认模式，兼顾质量与成本
- 深度：优先效果，放宽预算和上下文

- 交互要求：
- 每次会话可切换
- 支持团队策略设默认值
- 在结果页展示本次节省比例

### 6.2 功能二：上下文来源控制

- 用户可选上下文来源：
- 当前文件
- 当前模块
- 全仓库检索
- 历史对话（最近N轮）

- 默认策略：
- IDE 默认“当前文件+相关符号+最近3轮”
- Agent 默认“任务输入+工具摘要+最近2轮”

### 6.3 功能三：自动长对话压缩

- 触发条件：
- 历史对话累计 token 超阈值

- 执行动作：
- 对历史轮次做“事实摘要+决策摘要+待办摘要”
- 保留用户显式 Pin 的消息原文

### 6.4 功能四：回答长度档位

- 档位：
- 简短（默认）
- 标准
- 详细

- 行为：
- SDK 自动映射 `max_output_tokens`
- 输出模板根据档位自动收敛

### 6.5 功能五：预算提示

- 发送前给出 token 预估区间
- 超预算时提示“自动裁剪建议”
- 提供“一键低成本重试”

## 7. SDK 需求（统一中间件）

### 7.1 Budgeter

- 输入：场景类型、模式、租户策略、模型能力
- 输出：`input_budget`、`output_budget`、`step_budget`
- 规则：
- 超预算先裁剪上下文，再做压缩，最后降级模型

### 7.2 Context Packer

- 上下文项统一评分：
- `Score = 0.5*相关性 + 0.3*时效性 + 0.2*用户Pin权重`

- 裁剪顺序：
- 低分检索片段
- 低分历史轮次
- 冗长工具日志
- 仅在必要时触发深度摘要

- 永久保留项：
- 系统指令
- 当前用户问题
- 安全与合规约束

### 7.3 Compressor

- 历史摘要器：
- 产出三段结构：事实、决策、未决事项

- 工具输出压缩器：
- JSON 白名单字段保留
- 超长文本做句群摘要

- 代码上下文压缩器：
- 优先符号定义、调用链、diff，不默认传整文件

### 7.4 Cache Client

- 前缀缓存规范：
- 稳定内容前置：系统提示、项目规则、常驻文档摘要
- 动态内容后置：用户问题、即时检索结果

- 语义缓存规范：
- 仅缓存可复用问答与工具结果摘要
- 设置 TTL，按租户隔离命名空间

### 7.5 Model Router

- 路由分层：
- 分类、重写、摘要优先轻量模型
- 核心推理与最终回答使用主模型

- 路由策略可观测：
- 每次路由记录理由（成本/延迟/质量策略命中）

## 8. 网关需求（Higress 承接）

### 8.1 插件组合建议

- 必选：
- `ai-proxy`
- `ai-statistics`
- `ai-token-ratelimit`

- 推荐：
- `ai-cache`
- `ai-history`
- `ai-intent`

### 8.2 网关治理策略

- Token 配额：
- 按租户、应用、用户分级限额

- 历史治理：
- 默认开启 TTL，防止无限历史累积

- 缓存治理：
- 高命中路径优先接入缓存
- 定期清理低价值缓存 key

- 审计要求：
- 记录请求级 token 字段与路由信息
- 敏感字段脱敏后入日志

## 9. 应用研发最佳实践

### 9.1 Prompt 结构化

- 使用固定模板：角色、目标、约束、输出格式
- 不重复拼接相同系统说明

### 9.2 工具调用治理

- 工具输出只回填必要字段
- 原始超长日志不直接入模型上下文

### 9.3 上下文输入治理

- 检索 top-k 默认小值起步，按效果再增
- 默认短历史窗口，长历史交给摘要记忆

### 9.4 输出治理

- 强制 JSON schema 或固定模板
- 控制 `max_output_tokens`
- 失败重试优先低成本路径

## 10. 请求生命周期（标准链路）

1. 产品层读取用户模式与上下文开关
2. SDK Budgeter 计算预算
3. SDK Context Packer 评分并裁剪
4. SDK Compressor 按需压缩
5. SDK Cache Client 设置缓存 Hint
6. 请求进入 Higress，执行代理/缓存/限额/观测
7. 模型返回后记录 token 与质量埋点
8. 若超预算或失败，触发低成本重试策略

## 11. 数据埋点与看板

### 11.1 请求级字段

- request_id
- scene_type
- mode（极速/均衡/深度）
- model_id
- input_tokens
- output_tokens
- cached_tokens（若模型侧可返回）
- gateway_cache_hit
- semantic_cache_hit
- context_items_before
- context_items_after
- compression_triggered
- latency_ms
- success_flag

### 11.2 看板视图

- 成本看板：按租户/产品/模型聚合
- 质量看板：成功率、人工抽检、回退率
- 策略看板：裁剪命中率、缓存命中率、压缩收益

## 12. 灰度、回滚与风控

### 12.1 灰度策略

- Phase 1：10% 流量启用（均衡模式）
- Phase 2：30% 流量启用（加入自动摘要）
- Phase 3：80% 流量启用（加入语义缓存）
- Phase 4：全量

### 12.2 回滚条件

- 关键场景质量回退 > 2%
- P95 延迟劣化 > 20%
- 错误率显著升高

### 12.3 回滚动作

- 一键关闭压缩器
- 回退到“仅预算+缓存”策略
- 关闭语义缓存，仅保留精确缓存

## 13. 项目排期（12周）

- 打通统一埋点

- 上线网关缓存与 token 限额

### W6-W8：中期收益

- SDK 上线 Budgeter + Context Packer + Compressor

### W9-W12：高阶优化

- 全链路 A/B 与全量发布
