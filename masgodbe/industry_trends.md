# 业界主流大模型的底层架构真相：大家都是怎么玩的？

你的直觉非常敏锐！既然 DeepSeek 通过 MoE 架构把“计算和存储”分离，用 TP+EP 的策略在廉价卡上跑出了惊天动地的性价比，那 OpenAI、Google、Anthropic 这些硅谷巨头是怎么玩的？

答案是：**在逼近单机物理极限甚至跨越机房时，所有的顶级顶尖模型不约而同地走上了相似但又各有侧重的道路。**

以下是目前全球最顶尖几家 AI 实验室的底层玩法揭秘：

---

## 阵营 1：坚定的 MoE 拥趸（OpenAI & Google & Mistral）

**代表作：** GPT-4 (OpenAI), Gemini 1.5 Pro (Google), Mixtral 8x7B/8x22B (Mistral AI), DeepSeek V2/V3
**特点：** 极致的推理性价比（Token 价格极低），为了达到这个目的，他们把 **模型容量 (参数量)** 和 **每次计算所需要的算力 (激活参数量)** 脱钩了。

### 1. OpenAI (GPT-4 系列)

- **架构真相**：目前业界公认 GPT-4 并不是一个单一的稠密巨无霸，而是一个由 **8 个或 16 个 220B 规模的专家组成的巨大 MoE 模型**，总参数量高达 **1.76 万亿 (1.76T)**！
- **怎么部署的？**
  毫无疑问，这绝不可装在一台极其身上。OpenAI 在微软的 Azure 超算中心里，就是用的你刚学的 **TP（张量并行） + EP（专家并行） + PP（流水线并行）** 的终极 3D 混合并行组合！
  GPT-4 每次生成一个 Token，只激活 2 个专家（大约 280B 参数），剩下的 1.5T 参数都在“睡觉”。这也是为什么 GPT-4 能够扛住全球数亿人同时使用的原因——靠着牺牲大量内存（横跨极多甚至跨海机器）换来了极小的**单次激活计算代价**。

### 2. Google (Gemini 1.5 Pro)

- **架构真相**：Google 更是 MoE 架构的鼻祖级玩家（早年的 Switch Transformer 等）。Gemini 1.5 也是毫无疑问的 MoE 模型。
- **与众不同的玩法**：
  Google 拥有自己设计的硬件 **TPU (Tensor Processing Unit)** 和 **专用光纤网络（OCI / ICI）**。
  当你觉得跨机计算专家 (`All-to-All`) 走以太网太慢而痛不欲生时，Google 直接在数据中心铺设了比 NVLink 还变态的定制光纤和环形/胖树网络大拓扑！
  Google 的调度引擎（Borg，也就是 K8s 的爷爷）在部署 Gemini 时，其底层通信和专家分布也是基于刚才所讲的逻辑，不过他们是用更昂贵的硬件硬生生把通信延迟砸低了。这也是他们能做到恐怖的 200 万超长上下文（KV Cache 极其占显存）的底气！

## 阵营 2：极致力量的“暴力美学”（Anthropic & Meta）

**代表作：** Claude 3/3.5 Opus (Anthropic), LLaMA 3 400B (Meta)
**特点：** 在特定的时期和场景下，这几家坚持使用 **稠密模型（Dense Model）**。

### 1. Meta (LLaMA-3 400B)

- **架构真相**：纯粹的稠密模型（所有的参数，一个 Token 全部都要跑一遍计算）。
- **怎么部署的？**
  正如我们刚才讨论的，400B 的稠密模型装不进一台机器里！
  Meta 根本玩不了 MoE 的那种跨机专家调用。他们采用的部署方案是：**TP (张量并行，单机 8 卡) + PP (流水线并行，多台机器纵向接力)**。
  为了算一个词，整整 4000 多亿的权重，所有的 GPU 都要发光发热算一遍！这种运算极其**耗电、且吞吐率相较于 MoE 会低得多**。这就是为什么一直有传闻说 LLaMA-4 系列也要开始转向 MoE 的原因。如果不转，到了万亿参数，单纯靠 PP 不断加长流水线，机器间的相互等待（气泡）和单节点的故障率会把整个集群拖垮。

### 2. Anthropic (Claude 3.5 Sonnet / Opus)

- **架构真相**：Claude 3 家族一直是圈内的一个“黑盒”，但多方大佬（如 Jim Fan 等）猜测和推断，此前的 Claude Opus 大体量上依然偏向**庞大的稠密架构或极其有限数量内部路由的 MoE**。这也是为什么 Opus 虽然“极其聪明”，但它的 API 价格极其昂贵，而且生成速度曾经为人诟病的原因。
- **演进方向**：但随着最新 Claude 3.5 尤其是 3.5 Haiku 的发布（其速度之快令人咋舌），业界普遍认为 Anthropic 的推理后端架构也已经发生了翻天覆地的变化，极大概率也引入了更深度的路由/混合变体架构来降本增效。

## 你的大局观：为什么大家最终都殊途同归？

作为 AI Infra 的架构师，你必须要透过各大公司的各种黑话词汇（比如 VLLM, TensorRT-LLM, Megatron）看到那个不变的真理：

**1. 记忆芯片太贵太少 (Memory Bound)**
无论是哪家，想要模型变聪明，参数量就必须去到千亿、万亿级别。目前的单口 H100 即便有几十 GB 显存，对比起模型，也就是沧海一粟。**模型一定会被切散在茫茫多的服务器之间。**

**2. “算力”（计算核心能力）是溢出的**
GPU 矩阵乘法算得太快了，绝大多数时间都是在等待把参数或者 Token 从别的显卡 / 别的机器上搬运过来。不管是 OpenAI 用极其变态的 3D 并行，还是 Google 直接砸钱铺设 TPU 自定义网络光纤，核心都是**在解决“通讯问题”**。

这也就是为什么我最开始跟你说，你做 **TKE (K8s 调度)、CNI (高速容器网络)、CSI (并发存储加载)** 的这套组合拳是目前硅谷最热、最核心护城河的原因：

- **他们（OpenAI / DeepSeek）负责把模型写出来，把算法分身术写好。**
- **你（AI Infra 专家）负责在下面铺设无数条最高速的高铁轨道和精密的信号灯。如果你的轨道（网络和拓扑）不通畅，或者货车（CSI加载）跟不上，他们拥有再聪明的大脑也跑不起来。**
