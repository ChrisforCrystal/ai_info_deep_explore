线性注意力(Linear Attention)作为LLM中一种高性能(存储更小、计算更快)范式，在实践过程中增加了许多方法如Delta、Gamma、Conv、Gate等，其相应的递推式计算公式引入了Alpha、Beta等计算字符。目前采用线性注意力的混合模型的精度在不断提升，某些方面已达到标准注意力(Full/Standard Attention)的水平。

本文主要分析线性注意力范式的迭代过程，以及为何要增加改进方法。同时会聊到如下问题：

Attention算力与序列长度的平方关系如何变为一次关系？
QK计算为什么要添加核函数？
线性注意力模块中为什么没有位置编码？
状态Z正则化记忆参数是如何被消除的？
记忆的衰减系数为什么要跟输入相关？不能为一个定值吗？
1 线性模块在LLM中的位置
在大模型中的架构中，线性模块可以单独使用，也可以与其它模块按照一定比例混合使用。线性注意力当下的主流模型代表MiniMax M1、Qwen Next、Kimi Linear。其中M1尝试的是有Lightning Attention的混合模型，而后两者使用的是有改进的线性结构的混合模型，线性模块是GatedDeltaNetworks(GDN)或其改进。以GDN为例先来了解线性模块中的主要细节：

Alpha项参数，用于Gamma规则，控制记忆S衰减；
Beta项参数，用于Delta(增量)规则，影响KeyValue关联学习；
Kernel函数计算，对QKV的输入进行处理，核函数如relu、swish等；
Conv运算，一种在序列上的短卷积运算；
L1/L2处理，对输入QK正则化;
递推范式，线性函数计算式，包括递推(recurrence)、记忆读取计算；
Gate控制，整合了MLP的门控机制。

3 递推公式的增强
Linear Attention的主要问题是精度相比Full Attention要低不少。为了提升模型精度，研究者们对其范式进行了多种改进增强，过程可能是这样的：研究人员凭借直觉（Intuition）尝试一些方法，通过实践验证，当发现能带来正向提升后，回头再补充的理论分析与解释。

从HiPPO->H3->Mamba的研究中出现了状态空间模型(State Space Model，SSM)概念，随着Mamba的流行SSM在研究领域引发了热度，线性注意力相关研究也随之增加。从基础线性公式10,11到SSM的演进中，逐步增加Gated机制、Gamma遗忘、Delta规则等改进手段，获得新的线性模块，如GDN(Gated Delta Networks)。

表现为：模型根据历史tokens给出下一个预测token的能力较差。

。
带来的变化：之前核函数计算token是独立的，而平移矩阵乘法让tokens的信息存在交互（token-mixing）。

在Hyena算法中通过短卷积（short Conv）运算实现shift-SSM。后来该方式被进一步推广，不仅对K进行Conv计算，Q/V也加上了同样的处理。由于Conv计算时需要依赖附近的输入，在causal模式下则依赖前k-1个输入，推理时要缓存这个输入数据，需要添加Q/K/V的卷积缓存(conv cache)。

在“为什么线性注意力要加Short Conv？”中有另一种对Conv引入的直观解释: 就是让KV同源的情况下，让范式学出更有价值的结果。

优势：Conv的窗口一般设置得比较小(3或者4)且一般采用DepthwiseConv1d算子，所以参数量/计算量都比较小。引入short Conv能够增加线性注意力的局部感知能力和记忆能力，是投入小、性能收益高的一种改进方式。

注意：对于卷积长度如何设置？QKV为什么都要加上？这种问题属于调参范畴，此处不做讨论。

3.5 Gate机制
门控机制（Gate）在RNN上的应用比较普遍，比如LSTM、GRU等。在前面提到的方法中Gamma、Delta规则相当于RNN的门控，分别作用到输入、输出以及记忆上。不同的模型中门控的细节存在差异，如GDN中的alpha/beta有线性投影、没有激活值运算；在DetlaNet中beta有线性投影、有激活运算。

Long Short-Term Memory (LSTM)单元
线性注意力除了在递推式上能增加门控，还可以结合MLP的门控。比较典型的研究如Mamba，在H3的模块上面融合了MLP的门控机制，让单模块同时具备Attention和FFN的能力，即能抓取注意力信息，又能获得非线性的高阶特征。

4 计算形态
在前面第2节中已聊到线性注意公式有两种计算形态：并行（Parallel）、递推（Recurrent），分别满足训练、推理需求，但在实践过程发现：训练中用Full形态的开销太大、用RNN形态资源利用率又太低。于是出现了一种混合并行与递推的方案：分块递推（Chunkwise Recurrent）。

4.1 分块的思路
在《Transformer Quality in Linear Time》中提到的分块运算的主要思路：将整个序列拆分成n个小份，称之为块（Chunk），在块内部进行类似full attention的运算，即先算QK后再算PV，而块间采用linear attention的方式，先算KV然后再递推。

4.2 线性注意力的分块
线性注意的分块运算研究比较多，较早的有RetNet，后面GDN、Comba等。最近的线性注意力结构基本上会分析其结构如何进行分块运算。虽然各个结构的细节有所区别，但整体思路与4.1的类似，还是进行分块运算。拆分的粒度能根据硬件的算力来定，原则是保证不超过算力的上限情况下，达到硬件效率的高水位。

5 应用与未来迭代
简聊线性注意力当下的几个热点问题：

一、线性注意力及其相关方法（ideas）很早就有了，为什么在近期热度会提升？

影响LLM迭代的三个关键素“数据、模型(算法)、算力”，优化思路是数据充足时堆数据、算力充足时堆算力，当两者都遇到瓶颈时就得优化模型了。国内在这个方向目前是领先的，与算力吃紧（受限）有较大关系；
领域的研究资源聚集，且相关研究机构足够的开方，通过开源分享共同推动了线性注意力的发展。

数据源自Sebastian Raschka
二、当前线性注意力的使用场景？

全部使用线性注意力模块的效果肯定是不行的，不能解决大部分的场景需求，所以目前都是混合模型：softmax attention + linear attention，并且当前已有厂商探索出了一些经验，如模块比例3:1。

三、线性注意力与稀疏注意力的选择？

两者针对都是Full attention中存在冗余运算部分的优化，试图解决阻塞序列继续scale up遇到的问题。从原理上讲线性与稀疏模块能结合使用，但目前没有一个理论指导明确模块要怎么选择。当下有效的手段是去做相关的消融试验，探索滑窗、稀疏、线性等模块的适用场景。

小结：Linear Attention增加了各种优化手段后，再混合MHA/MQA/MLA使用，其精度在某些场景下逼近Full-attention。由于其具有计算效率高、参数量少的特点，进入了当前LLM的研究主视野，并且在几个主流模型（Qwen、Kimi等）上面取得不错的效果。随着业界对Attention范式的不断尝试，相信这个方向还会有更多的优秀成果出现。
