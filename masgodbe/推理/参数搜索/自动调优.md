TL;DR:模型调参是让模型取得更优效果的有效手段，也是每位模型 “炼丹师” 必备的核心技能。如何以更少的资源找到更优解，是机器学习领域持续探索的重要课题。即便进入大模型时代，超参搜索依然具备重要价值，有效解决诸多实际场景中的关键问题：

场景1：LLM 预训练的启动成本极高，如何合理配置 DP/TP/SP/PP/EP参数，才能显著提升训练效率？
场景2：当提出一种优质模型范式时，如何避免因参数配置不当导致范式效果未达预期，进而误判其核心价值？
场景3：推理场景中采样参数繁多、模型选型丰富，若需在 XX 资源约束下达成 XX 效果，如何快速匹配最优的模型与参数组合？
针对上述场景，参数调优仍是不可或缺的关键环节。自动超参搜索作为人工调参的重要补充，能够有效解决经验依赖、易陷入局部最优等问题，也能够节约系统资源(如LLM中的自适应并存策略)。本文将系统介绍超参搜索的常见方法及相关核心知识，同时分享一些我们团队探索的经验。

关于“多元自动搜索”方法的补充：

在模型参数量较小、技术范式尚处多元化的早期阶段，我们团队便已着手探索参数调优技术，从单任务自动调参搜索，逐步演进至多任务集群一体化搜索。多元搜索的核心概念，便是在这一系列探索与实践的过程中逐步凝练而成的。团队认为概念非常先进，所以我们在2019年申请了相关专利[1](高价值)，本人近期(2026年1月)收到美国专利版本的授权通知，这提醒了我可以摘取里面有意思点出来分享，详见2.4节。

1 概念与原理
模型训练的过程对于算法人员来说，是个典型的实验过程：提出想法->调整参数->实验->观测，重复循环。在机器学习中俗称“炼丹”，实验系统中的关键点：

优化指标(Objective)。选定观测指标，期望达到某一目标值；
搜索空间(Search Space)。由可尝试的参数范围构成的空间；
尝试(Trials)。根据设想或者分析，展开一系列的测试。
调优过程由算法工程师（俗称 “炼丹师”）设计实验、记录数据并分析，调优的速度与收益与工程师的经验高度相关。需要平衡：实验时间、计算资源、模型效果，三者之间的关系。

模型调参是指在模型范式确定的情况下，对可配置参数进行调整以期达到更好的收益。 根据业务场景调参一般分为：

效果/精度调优：通过调节参数，获得更好的模型输出指标。
性能调优：通过调节参数，使得模型在同规模资源下，运行效率更高。
算法工程师的调参多半是效果调优，平台工程师(AI Infra)则更偏向于性能调优。

1.1 一般模型
简单的调参系统（如下图所示），包括1. 待调节参数、2. 模型系统、3. 评价指标、4. 搜索器。

调参是一个黑盒搜索。通过搜索器调节模型参数，以最大化模型输出评价指标的收益。待调节的参数涵盖优化器超参、模型结构参数、数据裁剪参数等；评价指标即用户核心关注的量化指标，例如精度、召回率、准确率等。搜索器的实现形式分为两类：人工调参与自动搜索。

1.2 自动调参
自动调参过程是构建自动循环机制，结合历史搜索记录与搜索策略 / 算法，确定下一步的搜索参数，直至找到最优参数。在系统资源充足的情况下，可以并发搜索多组参数，如下图所示多资源(resource)多候选(candidates)的示例，该模式可显著缩短搜索过程的耗时。

搜索策略是指根据搜索的输出结果决定下次尝试的参数。不同的搜索策略对资源的消耗不一样，如下图所示，不同曲线表示不同参数在模型上的表现情况，搜索策略能够根据输出结果进行参数组的淘汰，控制资源消耗量。

搜索空间是指搜索算法能够探索的范围。搜索空间由一个以上的参数构成，参数的取值类型通常有选择(choice)、均匀值(uniform)、指数或者对数(exp/log)、正态分布(normal)。

不同类型取值的范围
自动调参既可作为独立步骤存在，也可嵌入运行系统，作为模型训练/推理的前置环节。自动调参的资源需求量并不需要等于实际运行时的资源量，可以少于或者大于实际运行量，这也给了AI Infra平台较大的发挥空间。

自动搜索作为训、推的前置步骤
自动搜索核心需解决的关键问题：是在资源消耗与收益之间找到最优平衡点。这里的资源通常指算力资源与训练时间。在精度搜索场景中，需在评估阶段使模型输出趋于稳定（即模型训练达到一定收敛程度），才能保证评价结果的可靠性，这无疑会消耗大量资源与时间；而在性能调优场景下，若总资源量固定，而自动搜索消耗过多资源，反而会挤占正常任务的资源配额。针对这一问题，目前已有诸多研究成果，例如通过早停技术、小资源验证、无资源模拟等方式降低搜索阶段的资源消耗，相关内容将在后续章节中详细展开。

1.3 大模型调参的特点
超参搜索与调参是机器学习领域中研究较早的方向，但不同阶段的调参工作具备差异化特征。步入大模型主导的当下，调参调优呈现出特征：

一、资源消耗巨大

在ResNet主导骨干网络（backbone）的阶段，模型参数量控制在百万（Million）级别，单张32G显存GPU即可完成模型的完整训练；进GPT系列模型阶段后，模型参数量跃升至十亿（Billion）级别，单台机器已无法承载大模型的训练任务。这意味着采用相同搜索策略时，资源消耗量可能呈指数级攀升。其中，基础大模型的预训练往往需要上千张GPU协同算力，这使得传统并发搜索模式面临严峻的资源瓶颈。

二、主流模型范式趋于统一

在小参数模型下，研究者会通过NAS(Neural Architecture Search)[2]技术来寻找新的模型范式的变形，达到更好的精度。NAS在当时取得了一定成果，如NASNet[3]、DARTS[4]、Evolution[5]等。

NAS的基本流程

当前主流应用场景以AI辅助类场景为主，例如聊天对话、内容生成等。相较于自动驾驶、物体检测等对精度要求严苛的场景，这类AI辅助场景对模型精度的要求相对更低，也为调参策略提供了差异化导向。

2 精度调优
精度调优的核心是关注模型的最终效果，当前行业主流采用“人工调优为主、自动调优为辅”的策略。下面将重点介绍几种常见的自动调优算法与工具。

2.1 开环搜索
开环搜索是指通过一系列测试实验筛选出结果最优的一组作为最终输出，且测试过程中产生的结果不会对后续实验流程产生反馈影响的搜索方式。

网格搜索（Grid Search）是从所有可能的参数取值中，均匀选取若干参数值，再通过参数排列组合生成待验证的试验案例。网格搜索的结果优劣取决于尝试次数（即网格密度），尝试次数越多（即网格密度越高），找到最优解的概率也就越大。

网格搜索适用于参数离散、参数量较少的场景。但网格搜索与人工调优存在明显差异，核心在于它缺乏对搜索环境的分析能力，无法形成完整的闭环调优过程。

随机搜索(Random Search)：将网格搜索中参数取值的遍历方式改为随机选取，即为随机搜索。随机搜索的资源消耗通常低于网格搜索，但受随机性本质影响，其搜索结果未必是最优解。

2.2 闭环搜索
闭环搜索是指采用多轮迭代机制的搜索方式，且每一轮的迭代输出会作为下一轮算法执行的参考依据。一般包含两类贝叶斯(Bayesian)和启发式(Heuristic)。对比[6]如下：

2.2.2 BOHB优化

Bayesian Optimization HyperBand(BOHB)[8]，一种高效的超参数优化算法，它结合了贝叶斯优化（BO）和连续减半算法（HyperBand）的优点，旨在以尽可能少的计算资源找到最优的超参数配置。HB通过“连续减半”（Successive Halving）机制，在有限的预算（如训练时间、数据量）下，动态淘汰表现差的超参数组合，保留有潜力的配置继续分配更多资源。

BO与HB算法融合
下图展示了BOHB算法的工作流程。此处设置最大预算max_budget = 9、最小预算min_budget = 1、衰减系数eta = 3，其余参数均为默认值。在该配置下，参数s_max = 2，因此算法将按{s=2、s=1、s=0、s=2、s=1、s=0……}的循环持续运行。在连续减半（SuccessiveHalving）流程的每个阶段（橙色方框标注部分），算法会筛选出排名前1/eta的参数配置，并为其分配更多预算以继续运行；该连续减半阶段会重复执行，直至本轮迭代结束。与此同时，算法会收集每次试验的参数配置、资源预算与最终指标数据，并基于这些数据构建以“预算”为核心维度的多维核密度估计（KDE）模型。该多维核密度估计模型将用于指导下一轮迭代的参数配置筛选工作。

2.2.3 TPE优化

Tree-structured Parzen Estimator(TPE)[9]，是一种基于树结构的贝叶斯优化算法，用于解决黑盒函数的全局最优化问题。传统贝叶斯优化（如使用高斯过程）直接建模P(目标值y | 参数x)，然后选择“最有希望”的x进行评估。TPE则另辟蹊径，它建模的是P(参数x | 目标值y)。简单来说，它试图回答的问题是：“当目标函数值很好（或很差）时，对应的参数x通常长什么样？”

与标准贝叶斯优化的区别：

特性 标准贝叶斯优化（如高斯过程） TPE
建模对象 P(y | x)（目标值给定参数的分布） P(x | y)（参数给定目标值的分布）
代理模型 高斯过程、随机森林等 两个核密度估计器（KDE）
采集函数 期望提升、上置信界等 密度比l(x) / g(x)
优势 提供预测和不确定性估计，理论完备 实现简单，在高维和条件空间表现好，早期收敛快
局限 计算开销随迭代次数增长快 对全局探索相对较弱，理论分析更复杂

2.3 搜索工具
索工具作为搜索算法与搜索流程的执行载体，核心工作内容包括：规范用户输入格式、构建标准化搜索流程、协同调度系统资源，为算法落地提供支撑。下面介绍几种常见工具：

2.3.1 Hyperopt

Hyperopt[12]是一个用于在复杂搜索空间进行串行和并行优化的Python库，其搜索空间可包含实值、离散和条件维度。Hyperopt是出现得比较早的超参搜索库，里面有多种搜索算法。优点是安装简单、使用方便。

2.3.2 Optuna

背景：作为AI Infra团队，我们早年间便已着手探索如何助力AI算法工程师以最低资源成本，高效探索最优算法解。但在当前的搜索过程中，输入与输出值均被视为离散指标，且通常采用单一算法进行评价；如前文提及的相关工具，往往还需预先定义具体的收益指标。

举个例子，下图左图为训练过程的Loss变化曲线：若仅以极值作为评价标准，算法可能选择曲线2，但会忽略模型的稳定性；右图为推理系统的GPU利用率曲线：若仅以累积值作为评价依据，可能选择曲线B，但会忽略集群的综合收益。

这不禁引发一个思考：收益指标能否实现多元化？能否综合多个指标的历史数据进行输出？

AI算法工程师在调优过程中，往往会参考多组数据与多个图表，进而得出下一步的调优决策。受到强化学习评价模型的启发，我曾尝试开发一款基于强化学习流程的参数调优工具，由模型自主定义评价指标。但很快便发现诸多问题：调优成本极高、搜参环境难以搭建，且模型易出现不收敛的情况。

而这类对个体用户而言极具挑战的问题，在AI平台或资源平台上得以解决，因为Infra平台能够在用户授权的前提下获取海量数据，助力评价模型持续迭代优化。由此，我们突发奇想提出了多元搜索的概念，旨在超大参数空间中完成参数调优。

多元搜索中搜索器的成长
多元搜索概念：指在由多维度数据构成的综合空间中开展搜索，核心是将输入视为多维连续变量，输出观测结果同样作为多维连续数据处理。搜索过程中需自适应控制搜索空间范围，可动态缩小或扩大范围，对参数关系进行灵活解耦或耦合，其自适应策略能够模拟专家的判断识别流程。

这里与一般搜索的区别：

空间范围：搜索过程中并非仅缩小所有变量的范围，而是在调整部分变量时，适度扩大其他变量的搜索空间，实现范围的动态适配。
耦合关系：传统搜索对各阶段参数采取离散化处理，例如先调整模型结构再优化超参；而多元搜索需灵活调整参数搜索顺序，打破固定流程，适配实际场景中的参数关联逻辑。
评估方式：打破单一指标的局限，兼顾多维指标与多类型指标（如各类图表数据），实现更全面的搜索效果评估。
我们最初的探索范围聚焦于模型相关参数，涵盖常见超参、结构参数、压缩策略等；后续将范围扩展至多模态数据（包括图片、表格、日志数据、经验文档等），调优系统可模拟专家行为，输出下一步调参方案。

性能调优主要是从节省算力资源角度开展的调优方式，选定评价指标如GPU利用率、MFU（模型算力利用率）、集群吞吐量，以及推理场景下的TTFT、TPOT等。在 LLM 性能调优场景中，受资源约束限制，催生出了小资源模拟搜索、无资源仿真等关键技术。鉴于 LLM预训练面临的挑战更为突出，资源瓶颈问题也更受行业关注，本节将重点介绍 LLM 训练过程中的自动并行策略调优。

3.1 常见方案
3.1.1 Galvatron

Galvatron[21]是由北京大学DAIR实验室开发的、用于Transformer大模型（尤其是大语言模型LLMs）的自动分布式训练系统。它的核心目标是自动化地为模型寻找并执行最优的混合并行训练策略，从而高效利用GPU集群资源，显著提升训练速度。特点：

细粒度逐层优化：系统可以为模型的每一层Transformer独立选择不同的并行策略组合，而非对整个模型使用单一策略，从而实现资源利用的最优化。PP被确定后，不同的layer都有自己独立的通信组。
智能搜索与决策[22]：基于对硬件（计算、通信）和模型（计算量、内存）的全面分析，通过成本建模和动态规划算法，自动搜索出在给定内存预算下吞吐量最高的并行方案。

工作流
一般的执行步骤（工作流）：

步骤1：采集硬件数据。通过HCCL-test采集集合通信的带宽，计算对通信的影响；
步骤2：采集计算和内存相关信息。通过单卡运行模型层，获取计算时间、显存占用量；
步骤3：搜索最佳策略。 决策树+动态规划搜索；
步骤4：执行模型运。根据推荐的参数运行模型。
步骤1和2在galvatron文件夹下生成json文件，记录profiling结果。galvatron/models/xxx/xxx.json

步骤3中的算法的设计假设（Takeaway）：

PP（流水线并行，Pipeline Parallelism）的通信量小，对通信带宽的要求较低；实践中通常优先执行PP切分，并将PP分组部署在跨度更大的设备上；
并行策略倾向于对设备进行均匀切分。例如，针对4卡GPU的2路DP（数据并行，Data Parallelism）场景，更倾向于将设备切分为两组各2卡的设备组，而非一组 1 卡、一组 3 卡的组合方式；
在可同时使用DP（数据并行）和SDP（分片数据并行，Sharded Data Parallelism）的场景下，仅使用SDP 在理论上具备更优性能，因此无需考虑同时启用DP与SDP。

不同流水线并行度（即8/4/2/1）下，适用于8块GPU的决策树示意图。我们选择其中一个分支来介绍如何使用该树描述候选的混合并行策略。根据要点#3的建议，我们移除了 1和 2策略，并在右侧展示了剩余的四种混合策略。所有这些决策树总共包含22种候选混合策略。
算法执行步骤：

针对指定模型，基于 Takeaway#1 和 Takeaway#2 的原则，优先通过 PP 将模型切分为多个阶段，同时将设备均匀且连续地划分为多个设备组。以 8 卡 GPU 场景为例，模型可被切分为 1/2/4/8 路 PP，分别对应规模为 8/4/2/1 的设备组；
每种 PP 切分方式对应一棵决策树与一个子搜索空间，该决策树的叶节点总数等于设备组规模，决策树的高度对应可用并行策略的数量；决策树的每一层仅可应用一种并行策略，且不同层之间的并行策略不可重复选用；
设计代价评估器（cost estimator）以估算计算与通信的耗时，并且对于模型中的每个 Transformer 层，均可采用不同的并行策略（TP、DP、SDP）组合。
3.1.2 Calculon

Calculon[23]是一种通过数学建模来寻找LLM模型的最优配置方案。能够根据设定的参数预估训练性能，进而能够对比多组参数配置，寻找最优解。

主要是进行算子粒度分析，侧重建模。包括计算量分析、显存占用分析、通信耗时分析。

在Calculon建模中有个Block概念。即多个算子在一起构成Block，分析block的量和耗时。比如：

计算量：block_fw_flops、block_fw_flops_time；
显存：block_weight_space；
通信：baseblock_fw_tp_size、block_fw_tp_time；
Calculon建模考虑的要素：

对PP流水的batch分析，考虑计算与通信掩盖：
每个Transformer模块（下图）由从 开始的模块编号和从 开始的微批次编号共同标识。每个处理器上的若干个连续模块被分组为块。每个“模块-微批次”组合的反向传播，在其所需数据就绪后立即被调度执行。流水线调度包含（a）所示的起始阶段和（b）所示的收尾阶段，两者之间的稳定阶段为简洁起见予以省略。收尾阶段展示了数据并行通信与其他Transformer模块的反向传播计算相重叠的过程。

考虑offload的时间影响。在当前 block 执行计算的同时，可并行完成下一个 block 的预取（prefetch）操作，以及前一个 block 计算结果的卸载（offload）操作。若前一个 block 的卸载耗时短于当前 block 的计算耗时，该卸载开销可被完全掩盖；反之，将产生部分无法掩盖的耗时。

Tensor offloading scheme modeled in Calculon
Calculon支持硬件参数设置(参考：https://github.com/calculon-ai/calculon/tree/main/systems)：

硬件计算单元：设置matrix、vector的TFlops大小，不同size、不同M\N\K对齐下的算力系数不同；
显存单元：显存大小与显存带宽，主要是HBM和shared memory；
网络传输单元：不同传输链路的速度。

Galvatron仿真仅需少量计算资源即可开展分析，其分析准确度由底层推算逻辑决定，在部分场景下可达到较高的精准度；

3.2 Infra平台侧解决方案
主流的机器学习平台上基本具备自动调优功能，比如AIAK[27]、PAI、DLRover[28]等。我们团队在这个方面也做了一些探索。并行策略的性能调优中，若采用全建模的抽象化处理无法精准复现驱动调度、硬件资源征用等底层复杂行为，易引发误差累积，采用Profile方案精度高但成本高。所以采用类似于Galvatron这类的方式，用小规模计算推导整性能的方式。下面介绍我们平台侧的一种解决方案。

3.2.1 系统架构

对于AI Infra平台而言，其核心优势在于拥有充足的用户侧数据，这些数据可提炼为经验知识，驱动系统实现自迭代优化。基于这一思路，系统整体架构如下所示，主要包含用户任务（user task）与搜索系统（Search System）两大核心模块：搜索系统可调度执行采集任务（profiling task），在资源配置层面，采集任务的资源规模与用户任务的资源规模越接近，最终的配置推荐效果越佳；数据库（database）则负责存储搜索过程产生的相关信息，以及用户任务的实际运行数据。

系统运行的基本步骤：

用户向搜索系统提交任务的相关信息，搜索系统基于小资源开销完成数据采集；
搜索系统基于采集到的信息，通过算法建模与计算，然后生成最优配置方案并下发至用户任务；
用户任务实际运行后，将实时性能数据反馈至搜索系统。

数据库的核心作用如下：

校准搜索推算指标：针对部分存在偏差的估算指标，可基于任务实际运行数据进行修正，提升搜索结果的准确性。
沉淀调优经验知识：随着模型范式逐渐收敛，以及历史数据的持续积累，部分场景下甚至可直接复用已验证的最优配置方案，无需额外执行采集任务。
3.2.2 算法运行流程

与 Galvatron 方案类似，我们以 PP 层为基本单元开展模块的数据采集工作；在大模块测试中，会根据搜索资源的大小裁剪网络(根据触发OOM调整)。
构建 block 级别的开销模型（cost model），例如模块端到端计算耗时、显存峰值等关键性能指标；场景包括：DP/TP/PP/CP/EP/VPP/Ulyssess/MBS/ZoRO/重计算/多流水等。
通过算法搜索最优配置，当资源充足时采用动态规划（DP）算法求解最优配置方案；当资源小时，开启黑盒搜索模式，如下图所示，搜索过程中最优值与历史数据相差较大时，会返回profiling阶段重新采样（用更大的资源），当training时输出差异与期望值差距较大时，会及时停止任务重新寻找最优值。

在cost model中，我们还纳入了不同模式下计算耗时的校准逻辑 ，包括图模式（Graph Mode）与即时执行模式（Eager Mode）的任务下发时间校准、GPU 与 NPU 等不同硬件的执行时间校准，以此保障推算结果的准确性。例举一组测试数据：

，resource 指标定义为搜索累积消耗的 GPU 卡时与训练总 GPU 卡时的比值，例如 resource=0.01 即代表搜索资源消耗占训练总资源的 1%。perf 指标参考专家调优水平，为训练吞吐的比值。考虑到“专家"能力存在差异，选用的历史最优值作为参考。搜索空间的可选配置均为离散枚举值，涵盖 pp、dp、tp、sp (cp)、ep、mbs、offload。

实践表明，仅需投入少量搜索资源，即可获得与专家调优持平甚至更优的结果。此外，在同模型的搜索场景下，若历史数据库中存在匹配记录，可直接命中复用，实现搜索资源的零消耗。进一步，在搜索空间有限且场景通用的情况下，可以利用空闲系统资源把最优值预先找出来。

总之，性能调优所节约的系统资源，可有效加速算法迭代进程，为AI技术创新提供动力。

import numpy as np
import matplotlib.pyplot as plt
from scipy.stats import norm
import warnings
warnings.filterwarnings('ignore')

# 1. 高斯过程核函数（径向基函数）

def rbf_kernel(x1, x2, length_scale=1.0, sigma_f=1.0):
"""径向基函数（RBF）核，也称为平方指数核"""
sqdist = np.sum(x1**2, 1).reshape(-1, 1) + np.sum(x2**2, 1) - 2 _ np.dot(x1, x2.T)
return sigma_f\*\*2 _ np.exp(-0.5 / length_scale\*_2 _ sqdist)

# 2. 高斯过程回归

class GaussianProcess:
def **init**(self, kernel=rbf_kernel, noise=1e-8):
self.kernel = kernel
self.noise = noise
self.X_train = None
self.y_train = None

    def fit(self, X, y):
        """拟合高斯过程（存储训练数据）"""
        self.X_train = X
        self.y_train = y

    def predict(self, X_test):
        """预测新点的均值和方差"""
        if self.X_train is None:
            # 如果没有训练数据，返回先验分布
            mu = np.zeros(X_test.shape[0])
            cov = self.kernel(X_test, X_test)
            return mu, cov

        # 计算核矩阵
        K = self.kernel(self.X_train, self.X_train) + self.noise * np.eye(len(self.X_train))
        K_s = self.kernel(self.X_train, X_test)
        K_ss = self.kernel(X_test, X_test)

        # 计算后验分布的参数
        K_inv = np.linalg.inv(K)
        mu_s = K_s.T.dot(K_inv).dot(self.y_train)
        cov_s = K_ss - K_s.T.dot(K_inv).dot(K_s)

        return mu_s, cov_s

    def sample_functions(self, X_test, n_samples=3):
        """从后验分布中采样函数"""
        mu, cov = self.predict(X_test)

        # 确保协方差矩阵是对称正定的
        cov = (cov + cov.T) / 2 + 1e-8 * np.eye(len(cov))

        # 采样
        samples = np.random.multivariate_normal(mu, cov, n_samples)
        return samples

# 3. 采集函数

class AcquisitionFunction:
@staticmethod
def expected_improvement(mu, sigma, best_y, xi=0.01):
"""预期提升（Expected Improvement, EI）"""
with np.errstate(divide='ignore'):
z = (mu - best_y - xi) / sigma
ei = (mu - best_y - xi) _ norm.cdf(z) + sigma _ norm.pdf(z)
ei[sigma == 0.0] = 0.0
return ei

    @staticmethod
    def probability_of_improvement(mu, sigma, best_y, xi=0.01):
        """提升概率（Probability of Improvement, PI）"""
        z = (mu - best_y - xi) / sigma
        return norm.cdf(z)

    @staticmethod
    def upper_confidence_bound(mu, sigma, kappa=2.0):
        """上置信界（Upper Confidence Bound, UCB）"""
        return mu + kappa * sigma

# 4. 目标函数

def objective_function(x):
"""具有多个局部最优值的测试函数"""
return -(np.sin(5*x) + 0.5*np.cos(10*x) + 0.1*(x-2)\*\*2)

def bayesian_optimization_detailed(): # 初始化参数
bounds = [-5, 5] # 搜索空间
n_iterations = 10 # 优化迭代次数
n_init_points = 3 # 初始随机采样点数

    # 创建网格用于可视化
    X_grid = np.linspace(bounds[0], bounds[1], 200).reshape(-1, 1)

    # 初始化数据
    X_observed = []
    y_observed = []

    # 高斯过程
    gp = GaussianProcess(kernel=lambda x1, x2: rbf_kernel(x1, x2, length_scale=1.0, sigma_f=1.0))

    # 存储每一步的信息用于可视化
    history = []

    # 步骤1: 初始随机采样
    print("Step 1: Initial Random Sampling")
    print("-" * 40)

    for i in range(n_init_points):
        x_new = np.random.uniform(bounds[0], bounds[1], 1)
        y_new = objective_function(x_new[0])

        X_observed.append(x_new[0])
        y_observed.append(y_new)

        print(f"  Point {i+1}: x = {x_new[0]:.3f}, f(x) = {y_new:.3f}")

    # 转换为数组
    X_obs = np.array(X_observed).reshape(-1, 1)
    y_obs = np.array(y_observed)

    best_idx = np.argmax(y_obs)
    best_x = X_obs[best_idx][0]
    best_y = y_obs[best_idx]

    print(f"\n  Initial best: x = {best_x:.3f}, f(x) = {best_y:.3f}")

    # 主优化循环
    print(f"\nStep 2: Bayesian Optimization Iterations (n={n_iterations})")
    print("-" * 40)

    for iteration in range(n_iterations):
        print(f"\nIteration {iteration+1}:")

        # 1. 拟合高斯过程
        gp.fit(X_obs, y_obs)

        # 2. 预测整个搜索空间的均值和方差
        mu, cov = gp.predict(X_grid)
        sigma = np.sqrt(np.diag(cov))

        # 3. 计算采集函数值
        ei = AcquisitionFunction.expected_improvement(mu, sigma, best_y, xi=0.01)
        pi = AcquisitionFunction.probability_of_improvement(mu, sigma, best_y, xi=0.01)
        ucb = AcquisitionFunction.upper_confidence_bound(mu, sigma, kappa=2.0)

        # 4. 选择下一个点（最大化EI）
        x_next_idx = np.argmax(ei)
        x_next = X_grid[x_next_idx][0]

        # 5. 评估目标函数
        y_next = objective_function(x_next)

        print(f"  Next point: x = {x_next:.3f}, f(x) = {y_next:.3f}")
        print(f"  EI value at x: {ei[x_next_idx]:.4f}")
        print(f"  GP prediction: mu = {mu[x_next_idx]:.3f}, sigma = {sigma[x_next_idx]:.3f}")

        # 6. 更新数据
        X_obs = np.vstack([X_obs, [[x_next]]])
        y_obs = np.append(y_obs, y_next)

        # 7. 更新最佳值
        if y_next > best_y:
            best_y = y_next
            best_x = x_next
            print(f"  New best! x = {best_x:.3f}, f(x) = {best_y:.3f}")

        # 存储历史信息用于可视化
        history.append({
            'iteration': iteration + 1,
            'x_next': x_next,
            'y_next': y_next,
            'mu': mu.copy(),
            'sigma': sigma.copy(),
            'ei': ei.copy(),
            'pi': pi.copy(),
            'ucb': ucb.copy(),
            'best_x': best_x,
            'best_y': best_y,
            'X_obs': X_obs.copy(),
            'y_obs': y_obs.copy()
        })

    # 最终结果
    print(f"\nStep 3: Final Results")
    print("-" * 40)
    print(f"Best solution found: x = {best_x:.4f}, f(x) = {best_y:.4f}")

    # 理论最优值（对于这个简单函数）
    x_test = np.linspace(bounds[0], bounds[1], 1000)
    y_test = objective_function(x_test)
    true_opt_idx = np.argmax(y_test)
    true_opt_x = x_test[true_opt_idx]
    true_opt_y = y_test[true_opt_idx]

    print(f"Theoretical optimum: x = {true_opt_x:.4f}, f(x) = {true_opt_y:.4f}")
    print(f"Error: {abs(best_x - true_opt_x):.4f}")

    return history, bounds, X_grid
